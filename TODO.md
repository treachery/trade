# TODO / 路线图

## 背景痛点
当前收到入场信号后，**不知道该用哪种策略入场**——根本原因是不知道标的"当前处于什么形态"。
现有"形态推荐策略 + 一键寻优"两组对比已是雏形，但形态判断是基于简单的涨跌幅/高低点规则，不够精准，也没有"持仓后持续校准"的能力。

---

## 需求一：相似K线检索驱动的策略选择（Pattern Matching / Analog Forecasting）

**目标**：拉取标的近一年走势，在历史数据库里找出"最相似"的若干段K线，截取这些相似段的**后半段**跑策略分析，挑出在相似情形下**胜率/收益最高**的策略作为入场策略。

**拆解**：
1. 构建历史K线片段库：对数据库内全部标的，用滑动窗口切出大量历史片段（如每段 250 日 ≈ 1 年）。
2. 相似度检索：把当前标的近一年走势与库中片段比对，取 Top-K 最相似。
3. 后半段回测：截取这些相似片段的"后半段"（即历史上"相似形态之后"发生了什么），在其上跑全部候选策略。
4. 策略选优：统计每种策略在这 K 段后半段里的平均收益/胜率/回撤，选最优者作为该标的的入场策略。

**关键技术点**：
- 序列归一化（去量纲）：z-score 标准化或首值归一，让不同价位的标的可比。
- 相似度度量：欧氏距离 / 相关系数 / DTW（动态时间规整，容忍时间轴伸缩）。
- 检索加速：片段量大时需向量化 + 近似最近邻（FAISS/Annoy）。

## 需求二：持仓后持续迭代校准（Online Tracking / Regime Drift Detection）

**目标**：入场基于"相似参考K线"，但入场后要**持续监控**：
- 每日轮询，检查标的实际走势是否仍与"当初选取的参考K线后半段"拟合；
- 若拟合度下降（形态漂移），重新检索最拟合的参考片段、重选预测与适配策略；
- 据此动态调整持仓/止盈止损/换策略。

**拆解**：
1. 拟合度跟踪：每日计算"入场以来实际走势" vs "参考片段对应段"的偏离度（残差/相关性衰减）。
2. 漂移检测：偏离超过阈值 → 判定形态漂移，触发重检索。
3. 重适配：重新跑需求一的流程，更新参考片段与推荐策略；必要时发出"换策略/减仓/离场"提醒。
4. 记录轨迹：保留每次重适配的历史，便于复盘。

## 我的实现思路（分阶段，先简单后复杂）

### 阶段 0（已具备）
- 规则式形态判断（长牛/长熊/牛熊/熊牛/震荡）+ 形态映射策略 + 一键寻优，两组对比选优。

### 阶段 1：相似K线检索（MVP，纯 numpy，无需额外依赖）
- 片段库：对已缓存的 A500/港美等标的，按 250 日窗口、步长 ~20 日切片，存 `data_cache/pattern_lib.parquet`（或 npy）。
- 特征：每段取"对数收益率序列"并 z-score 归一（消除价位与波动量纲）。
- 相似度：先用相关系数 / 欧氏距离做粗排，Top-K 候选再用 DTW 精排。
- 后半段回测：对每个相似片段，取其"后 N 日"原始价格序列，跑候选策略库，汇总胜率/年化/回撤。
- 选策略：用现有评分口径（卡玛/夏普/年化/资金效率 × 回撤/超短惩罚）对"相似情形后半段"的平均表现排序，取 Top1。
- 产出：给当前标的一个"基于历史相似情形"的推荐策略 + 置信度（相似度均值、样本数）。

### 阶段 2：检索加速 + 质量提升
- 片段量上来后引入 FAISS/Annoy 近似最近邻；DTW 用 `fastdtw` 或 `tslearn`。
- 相似度叠加多维特征：收益序列 + 波动率 + 成交量形态 + 趋势斜率。
- 引入"形态聚类"（K-Means / 层次聚类 on 归一化片段）做离线形态字典，在线只需匹配到簇。

### 阶段 3：持仓后在线校准（轮询任务）
- 复用已有的"每日收盘后自动扫描"调度，新增"持仓校准"子任务。
- 维护持仓表：{标的, 入场日, 参考片段ID集合, 推荐策略, 入场以来实际走势}。
- 每日计算拟合度（实际 vs 参考片段对应段的相关性/残差），低于阈值触发重检索 + 重适配，并通过 webhook 提醒"形态漂移/建议换策略"。

## 业界方案 / 参考

1. **Analog Forecasting / Pattern Matching（类比预测）**
   - 思路与本需求一致：找历史相似情形，用"之后发生了什么"做预测。气象、宏观、量化都在用。
   - 关键：相似度度量 + 样本充分性 + 防过拟合。

2. **DTW（Dynamic Time Warping，动态时间规整）**
   - 时间序列相似度的经典方法，容忍时间轴伸缩/错位，比欧氏距离更鲁棒。
   - 库：`tslearn`、`fastdtw`、`dtaidistance`。

3. **Matrix Profile（STUMPY 库）**
   - 高效计算时间序列"motif（重复模式）/discord（异常）"，可做大规模相似子序列检索，比朴素 DTW 快很多。
   - 非常适合"在历史里找与当前最像的片段"，工程上成熟。

4. **向量化检索（FAISS / Annoy / HNSW）**
   - 把每段K线编码成向量（手工特征或自编码器 embedding），用近似最近邻在百万级片段里毫秒检索。

5. **表示学习 / 时序embedding**
   - 自编码器、对比学习（如 TS2Vec）、TimeGPT 等把K线片段编码成向量再检索，鲁棒性强但工程复杂。

6. **Regime Detection / Change-Point Detection（市场状态/漂移检测）**
   - HMM（隐马尔可夫，状态=牛/熊/震荡）、贝叶斯变点检测、CUSUM、`ruptures` 库。
   - 对应需求二的"形态漂移检测"。

7. **K线形态识别**
   - TA-Lib 的经典形态函数（如 CDL* 系列）做辅助特征。

### 推荐落地组合（务实路线）
- 检索：**STUMPY (Matrix Profile)** 找相似子序列 + **z-score 归一** + 必要时 **DTW 精排**。
- 漂移：**`ruptures` 变点检测** 或简单的"滚动相关性 + 阈值"。
- 加速：片段多了再上 **FAISS**。
- 全部可纯离线 + 本地缓存，契合本项目"数据缓存到本地、轮询扫描"的现有架构。

## 风险与注意
- **过拟合 / 幸存者偏差**：相似片段后半段表现好≠未来一定好，需要足够样本量 + 置信度展示 + 防止只用少数极端相似片段。
- **未来函数（look-ahead）**：检索时只能用"当前及之前"的数据，参考片段的后半段只能来自历史，绝不能引入未来信息。
- **算力**：全市场滑窗片段量很大，需缓存 + 增量更新 + 近似检索。
- **相似度的"形似≠神似"**：价格形似但驱动逻辑不同，建议叠加多维特征并展示置信度，作为辅助而非唯一依据。
