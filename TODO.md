## 代码中的潜在 Bug 和不合理设计

### 1. 数据增量更新逻辑可能有问题
- **位置**：`backtest/data.py` 中的 `_need_refresh()` 函数
- **问题**：判断逻辑是"缓存最新日期距今 ≥ 2 个自然日"就更新，但没有考虑节假日和非交易日
- **建议**：使用交易日历，只比较交易日

### 2. PE 数据缓存可能不一致
- **位置**：`backtest/data.py` 中的 `load_pe()` 和 `load_valuation_pe()`
- **问题**：PE 数据来自不同源（乐咕、富途、WorldPERatio），缓存没有版本管理
- **建议**：增加数据源版本标识，数据源变更时自动刷新缓存

### 3. 前端没有错误处理
- **位置**：`static/pattern.js`
- **问题**：API 请求失败时，前端错误信息可能不够友好
- **建议**：增加全局错误处理和用户友好提示

### 4. 相似检索性能可能不足
- **位置**：`pattern_matching/retrieval.py`
- **问题**：使用 numpy 矩阵运算，但当候选池达到百万级时可能内存不足
- **建议**：考虑使用 FAISS 或 hnswlib 进行向量检索优化

### 5. 策略评估没有考虑交易成本
- **位置**：`pattern_matching/evaluator.py`
- **问题**：策略评估计算收益时没有扣除交易成本
- **建议**：在 `evaluate_strategy_on_fragments()` 中增加交易成本计算

### 6. 没有参数化测试
- **位置**：整个项目
- **问题**：没有单元测试或集成测试
- **建议**：引入 pytest，为核心模块增加测试用例

## 技术债务和重构建议

### 1. `app.py` 过于臃肿
- **当前状态**：`app.py` 有 84KB，包含所有路由和业务逻辑
- **建议**：
  - 将路由按模块拆分到 `routes/` 目录
  - 或使用 Flask Blueprint 进行模块化

### 2. 配置管理混乱
- **当前状态**：配置分散在代码和环境变量中
- **建议**：使用统一的配置文件（如 `config.py` 或 `config.yaml`）

### 3. 前端没有构建流程
- **当前状态**：前端是原生 JS + 内联 CSS，没有使用现代前端工具链
- **建议**：考虑引入 Vite + React 或 Vue 进行前端重构（可选，取决于项目规模）

### 4. 数据缓存没有清理机制
- **当前状态**：`data_cache/` 目录会无限增长
- **建议**：增加缓存清理策略（如只保留最近 N 天的数据）

### 5. 日志系统不完善
- **当前状态**：使用 print 语句，没有统一日志系统
