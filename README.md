# 股票量化研究平台

一个轻量、自用的 **A 股 / 港股 / 美股** 量化研究平台，包含三大核心模块：

| 模块 | 核心能力 | 入口 |
|---|---|---|
| **① 交易策略回测** | 可组合策略库 + 一键寻优 + PE 估值仓位管理 | 首页 `/` |
| **② 市场信号监测** | 全市场信号扫描 + 订阅推送（Webhook/邮件） | `/trade-notify` |
| **③ K线模式匹配** | 相似片段检索 + 策略评估排序 + 可解释性研究 | `/pattern-research` |

> 在线体验：http://49.233.175.122（腾讯云轻量服务器，Nginx + gunicorn + Docker）

## 快速开始

### 本地开发

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动（或双击 run.bat）
python app.py
# 浏览器打开 http://127.0.0.1:5000
```

默认加载 **沪深300（sh000300）· 近 10 年**，自动跑一次回测。

### 港股/美股估值数据依赖（可选）

港股和美股个股的 PE 估值数据依赖 **富途 OpenD**，如需该功能：

1. 下载并安装 [富途 OpenD](https://www.futunn.com/download/openAPI)
2. 启动 OpenD，确保本地端口 `11111` 可访问
3. 在环境变量中配置 `FUTU_OPEND_HOST=127.0.0.1:11111`
4. 如未配置，系统会自动退化（港股/美股估值相关功能不可用，回测仍正常）

> ⚠️ 富途 OpenD 需要富途账号登录，仅用于获取估值数据，行情数据仍走 yfinance。

## ① 交易策略回测

> 入口：首页 `/` — 可自由组合入场/出场策略，在线查看买卖点与资金曲线。

### 亮点功能

**可组合策略库（5 入场 + 5 出场，支持 AND / OR）**

| 入场策略 | 出场策略 |
|---|---|
| 双均线金叉（默认 MA50/200） | 跌破均线（默认 MA20） |
| 唐奇安通道突破（创 N 日新高） | 吊灯 ATR 止损 |
| 均线多头排列（MA5>10>20>60） | 移动止盈（最高点回撤 %） |
| MACD 零轴上金叉 | 唐奇安下轨（创 N 日新低） |
| 量价突破（新高 + 放量） | 双均线死叉 |

- 入场/出场各可多选，并独立选择 **任一(OR)** 或 **同时(AND)** 逻辑
- 入场在「组合条件由假变真」当天买入；出场在「组合条件成立」当天卖出，卖出原因会标明是哪条触发

**一键寻优（3249 种组合批量回测）**

- 选好标的与区间，点 **「⚡ 一键寻优」**，后台跑 **全部 3249 种结构组合**（预计算指标，约 3 秒）
- 按**基金式综合评分**排 Top10：
  $$\text{综合分}=0.35\times\text{夏普}+0.25\times\text{卡玛}+0.25\times\text{超额年化}+0.15\times(\text{回撤越小越好})$$
- 自动剔除交易 < 10 笔的样本（避免过拟合噪声）；并按**最小原则去重**（绩效完全相同的组合只保留最简版本）
- 结果可点「载入」直接回填表单并出图

**市场环境最佳策略（牛 / 熊 / 牛熊）**

页面顶部一键分别在 **牛市(2019-01~2021-02)**、**熊市(2021-02~2024-02)**、**一轮牛熊(2019-01~2024-02)** 区间寻优，给出各自最佳组合并可直接载入。

**PE 估值仓位管理（含融资杠杆 + 按日计息）**

- **PE 百分位定仓**：`仓位 = (1 − PE百分位) × 2`，钳制到 `[最低0.5, 最高2.0]`。便宜时加杠杆（>1 部分为融资），贵时减仓
- **融资计息**：仓位 > 1 的借款部分按**年化融资利率（默认 6.99%）按日计提利息**
- **PE 上升逐步卸杠杆**：持仓中 PE 百分位每上升 N 个点（默认 10）就按新仓位减仓，逐步降杠杆
- 也可切换为**固定仓位**

**交易明细与可视化**

- K 线图标注每个**买点 / 卖点 / 卸杠杆减仓点**，可缩放
- 资金曲线 vs 买入持有基准对比图
- 概要卡片：总回报、年化、超额、夏普、卡玛、最大回撤、胜率、融资利息、交易手续费、卸杠杆次数等
- 逐笔明细表：买卖日期价格、**仓位、买入/卖出金额、融资利息、手续费、盈亏金额、收益%**，卸杠杆减仓作为可折叠子行展示；按时间从近到远排序

### 使用方法

1. 输入标的代码（支持 A 股/港股/美股/指数）
2. 选择时间区间、复权方式、初始资金
3. 勾选入场/出场策略，选择组合逻辑（OR/AND）
4. 点「运行回测」→ 查看 K 线图、资金曲线、交易明细
5. 点「一键寻优」→ 查看 Top10 最优策略组合

## ② 市场信号监测

> 入口：`/trade-notify` — 全市场扫描买卖信号，支持订阅推送。

### 亮点功能

**全市场信号扫描**

- 预置 5 个入场信号 + 5 个出场信号，覆盖均线、通道、MACD、量价等经典指标
- 支持扫描**中证 A500 成分股、标普 500 成分股、港股大市值、行业指数、宽基指数**
- 扫描结果包含：信号类型、触发日期、当前 PE 分位、建议仓位

**自定义股票池**

- 支持手动输入自定义股票列表
- 支持按分类快速选择（A500 / 标普500 / 港股 / 行业指数 / 宽基指数）

**订阅推送（Webhook + 邮件）**

- 配置 Webhook URL，信号触发时自动 POST 推送（兼容企业微信/钉钉/飞书）
- 配置 SMTP 邮件，信号触发时自动发送邮件通知
- 支持设置推送频率（每日盘后 / 实时）

**信号质量评分**

- 基于历史回测表现对每个信号打分
- 综合评分 = 夏普 × 0.35 + 卡玛 × 0.25 + 超额年化 × 0.25 + 回撤 × 0.15
- 优先推送高分信号

### 使用方法

1. 进入 `/trade-notify` 页面
2. 选择要扫描的股票池（A500 / 标普500 / 自定义）
3. 设置扫描参数（回溯天数、信号类型）
4. 点「开始扫描」→ 查看信号卡片列表
5. 配置推送方式（Webhook URL 或 SMTP 邮件）
6. 点「保存订阅」→ 系统每日盘后自动扫描并推送

## ③ K线模式匹配（动态情景匹配）

> 入口：`/pattern-research` — 不预测未来，而在历史中找"当下走势最像的片段"，看这些片段后续表现，辅助决策。

### 核心算法

| 步骤 | 算法 | 说明 |
|---|---|---|
| 信号侦测 | 规则引擎 | 10+ 种技术指标（突破、金叉、回踩等） |
| K线切片 | 滑动窗口 | 250 日窗口、步长 20 日、对数收益率、z-score 标准化 |
| 相似检索 | 矩阵化 KNN | 四级递进过滤（5→10→20→30日），numpy 矩阵运算 |
| 策略评估 | Bootstrap | 1000 次重采样置信区间，多目标稳健评分 |

### 亮点功能

**逐级递进检索（第6层）**

- 四级形态过滤（5日 → 10日 → 20日 → 30日）
- 矩阵化 KNN 相似度计算，检索性能优异
- 支持跨股票检索（不局限于单一标的）

**策略评估排序（第7层）**

- 在相似历史片段上评估 10 个候选策略
- Bootstrap 置信区间，多目标稳健评分
- 样本不足时自动标注"置信度低，仅供参考"

**可解释性可视化**

- 前端 Canvas 渲染 K 线图，可叠加/隐藏相似片段
- 展示相似片段列表（含相似度、后续 5/10/20 日收益）
- 策略评估排名面板（含样本数、置信度、尾部风险）

### 使用方法

1. 进入 `/pattern-research` 页面
2. 选择标的 + 分析日期 + 参考标的池（默认 A500 成分股）
3. 点「开始分析」→ 自动侦测信号 → 跨股票检索相似 K 线
4. 查看相似片段列表（可按相似度/收益排序）
5. 勾选感兴趣的相似片段，点「策略评估」
6. 查看推荐策略排名（含样本数、置信度、尾部风险）

## 数据来源与缓存

### 数据来源

| 市场 | 行情数据 | PE 估值数据 |
|---|---|---|
| **A股** | akshare（东方财富） | akshare / 乐咕 / 百度 |
| **港股** | yfinance（雅虎财经） | futu-api（需 OpenD） |
| **美股** | yfinance（雅虎财经） | futu-api（需 OpenD）/ WorldPERatio |

### 缓存机制

- 首次拉取某标的会联网，之后读本地 `data_cache/` 缓存
- **自动增量更新**：缓存最新日期距今 ≥ 2 个自然日（即可能有新交易日）时，自动联网补到最新；非交易日 / 当天未收盘拉不到时静默用缓存
- 联网失败自动重试并回退本地缓存

## 性能优化

### 服务端 LRU 缓存 + 启动预热

- 回测/寻优结果按参数缓存（最多 100 组，LRU 淘汰），相同参数再次请求直接返回，不重新计算
- **启动预热**：服务启动后在后台异步预热 5 组默认数据（默认回测 + 一键寻优 + 牛/熊/牛熊三段），新用户打开页面即秒出
- 缓存命中不计入限流

### 客户端缓存 + 载入动效

- 运行回测 / 一键寻优 / 市场环境最佳策略 时，若参数未变则不重复请求后端，直接用上次结果渲染（带 1 秒载入动效，避免假数据观感）

### Nginx 反向代理 + gzip 压缩

- Nginx 监听 80 端口代理到 gunicorn(5000)，开启 gzip 压缩（266KB JSON → 86KB），大幅减少公网传输
- 正确传递客户端真实 IP（`X-Real-IP`），支持 Cloudflare（`CF-Connecting-IP`）

### IP 限流

- 同一 IP 多窗口限流：每秒 1 次 / 每分钟 10 次 / 每小时 100 次 / 每天 1000 次，超限返回 429 并弹窗提示

## 标的代码格式

- **A股个股**：直接 6 位代码，如 `600519`（贵州茅台）
- **A股指数**：加 `sh`/`sz` 前缀，如 `sh000300` 沪深300 / `sh000001` 上证指数 / `sz399001` 深证成指 / `sz399006` 创业板指
- **港股**：指数 `hkHSI` 恒生 / `hkHSTECH` 恒生科技 / `hkHSCEI` 恒生国企；个股 `hk`+5位，如 `hk00700`（腾讯）
- **美股**：指数 `us.SPX` 标普500 / `us.NDX` 纳指100；个股 `us`+东财代码，如 `us105.AAPL`（苹果）/ `us105.TSLA`（特斯拉）

## 部署（Docker + Nginx / 腾讯云轻量服务器）

> 在线实例：http://49.233.175.122（腾讯云轻量服务器 · 北京 · Ubuntu）

### 架构

```
公网:80 (Nginx)  ──proxy──>  127.0.0.1:5000 (Docker/gunicorn)
                                  │
                        /root/trade_data_cache (volume)
```

- Nginx 监听 80 端口，gzip 压缩 + 反代到 gunicorn(5000)
- Docker 容器仅监听 `127.0.0.1:5000`，不对外暴露
- `data_cache/` 通过 volume 挂载到宿主机 `/root/trade_data_cache`，重启不丢行情/PE 缓存
- 环境变量通过 `--env-file /root/trade.env` 注入（GITHUB_TOKEN / SMTP 等）

### 首次部署

#### 1. 准备环境变量文件

在服务器 `/root/trade.env` 写入（**勿提交到 git**）：

```bash
GITHUB_TOKEN=github_pat_xxx
GITHUB_REPO=treachery/trade
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_SSL=1
SMTP_USER=you@gmail.com
SMTP_PASS=应用专用密码
SMTP_FROM=you@gmail.com
```

#### 2. 构建 Docker 镜像

```bash
docker build -t trade:latest .
```

> `.dockerignore` 已排除 `data_cache/`、`.git/`、`__pycache__/` 等，构建上下文仅含源码（约 1MB）。

#### 3. 运行容器

```bash
docker run -d --name trade --restart unless-stopped \
  -p 127.0.0.1:5000:5000 \
  -v /root/trade_data_cache:/app/data_cache \
  --env-file /root/trade.env \
  trade:latest
```

#### 4. Nginx 反向代理

```nginx
server {
    listen 80;
    server_name _;
    # 如使用 Cloudflare，取消下行注释以获取真实访客IP
    # set_real_ip_from <cloudflare-ip-ranges>;
    # real_ip_header CF-Connecting-IP;

    gzip on;
    gzip_types application/json text/css application/javascript;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 120s;
    }
}
```

- 防火墙仅开放 80（HTTP）和 22（SSH），5000 端口不对外暴露
- 国内服务器拉取镜像建议配置 Docker 镜像加速器（如 `mirror.ccs.tencentyun.com`）

### 更新部署（CodeBuddy Lighthouse 集成）

代码提交后，通过 CodeBuddy 的 Lighthouse 集成一键更新云上服务，**不在服务器上拉代码，而是上传源码到服务器打镜像**：

#### 步骤

1. **上传项目文件**：调用 `deploy_project_preparation`，将本地项目目录上传到服务器 `/root/trade_<时间戳>/`
   - `.dockerignore` 已排除 `data_cache/`（119MB），上传仅含源码（约 1MB），几秒完成
   - 旧的上传目录会累积，定期清理：`rm -rf /root/trade_2026*`

2. **构建新镜像**：在服务器上执行
   ```bash
   docker build -t trade:latest /root/trade_<时间戳>/
   ```

3. **重启容器**：
   ```bash
   docker stop trade && docker rm trade
   docker run -d --name trade --restart unless-stopped \
     -p 127.0.0.1:5000:5000 \
     -v /root/trade_data_cache:/app/data_cache \
     --env-file /root/trade.env \
     trade:latest
   ```

4. **验证**：
   ```bash
   curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:5000/   # 期望 200
   docker logs trade --tail 20                                       # 查启动日志
   ```

5. **清理旧镜像**（可选）：
   ```bash
   docker image prune -f    # 清理 dangling 镜像
   ```

## 开源股票/量化策略平台调研对比

| 平台 | 语言 | 数据 | 接入方式 | 策略支持 | 历史回测 | 适合 |
|---|---|---|---|---|---|---|
| **Backtrader** | Python | 需自接(yfinance/akshare/CSV) | 库 import | 事件驱动，指标/多品种/多周期完善 | 强，支持滑点/手续费 | 个人/研究，最流行通用回测 |
| **Backtesting.py** | Python | 自接 | 库 import | 轻量，单品种为主 | 强，内置可视化 | 快速验证单策略（与本项目最接近） |
| **VectorBT** | Python | 自接 | 库 import | 向量化、超快、可大规模寻优 | 极快 | 批量回测/因子研究 |
| **Zipline(-reloaded)** | Python | bundle(美股为主) | 库 import | Quantopian 同源 | 强 | 学术/美股 |
| **vn.py** | Python | 实盘接口丰富 | 框架+GUI | 回测+实盘全栈 | 有 | 国内实盘交易 |
| **RQAlpha(米筐)** | Python | A股内置较好 | 框架 | A股股票/期货 | 强 | A股策略研究 |
| **QUANTAxis** | Python | A股全市场 | 框架 | 数据+回测+模拟 | 有 | A股数据中台 |
| **Qlib(微软)** | Python | A股/美股+数据集 | 框架 | 机器学习量化 | 有 | AI/因子选股 |
| **QuantConnect/LEAN** | C#/Python | 云端海量(全球) | 云平台/本地 | 全栈 | 强 | 多资产、云端 |
