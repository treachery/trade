# BE-033 相似检索 API

- **类型**：后端/API
- **优先级**：P3
- **状态**：待办

---

## 1. 需求目标

为前端提供完整相似K线数据，避免前端拼凑。

## 2. 需求范围

- 入参：symbol/as_of_date/reference_universe/top_k
- 出参：query_kline、similar_windows、stage_logs、diagnostics
- 后端日志包含 query_id 和耗时

## 3. 依赖关系

- `BE-032`

## 5. API / 函数设计

```http
POST /api/pattern/retrieval/query
{
  "symbol":"600519",
  "as_of_date":"2026-06-29",
  "reference_universe_id":"ref_a500",
  "top_k":10
}
```

## 6. 数据结构示例

```json
{
  "query_kline":{"dates":[],"ohlc":[]},
  "similar_windows":[
    {"symbol":"000858","name":"五粮液","anchor_date":"2023-06-12","kline_before":[],"kline_after":[]}
  ],
  "stage_logs":{"t20":{"input":12000,"output":100}}
}
```

## 7. 验收标准

- [ ] K线字段可直接用于 ECharts candlestick
- [ ] similar_windows 每项必须有 symbol/name
- [ ] 失败时前端可展示 error + diagnostics
