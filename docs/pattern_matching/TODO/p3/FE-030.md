# FE-030 相似K线研究页 MVP

- **类型**：前端
- **优先级**：P3
- **状态**：已完成 ✅

---

## 1. 需求目标

用户能清楚看到当前标的K线、相似标的是谁、相似时间段、以及相似片段后半段走势。

## 2. 页面 / 组件范围

- 相似K线研究页
- K线主图
- 相似片段列表
- 页面日志/请求状态区域
- 策略评估入口

## 3. 按钮与交互

- **开始分析**：调用相似检索API并展示页面日志
- **勾选叠加**：控制某个相似片段是否显示在主图
- **前瞻走势**：显示/隐藏该片段锚点后的历史走势
- **仅看当前标的**：隐藏所有相似片段
- **展开检索链路**：查看 t20/t40/t60/env 候选收缩过程

## 4. 展示数据

- `query_kline`：当前标的选中时间段 OHLC K线
- `similar_windows[].symbol/name`：相似股票代码和名称
- `similar_windows[].similar_period`：相似片段时间段
- `similar_windows[].kline_before`：相似片段前半段K线
- `similar_windows[].kline_after`：锚点后历史走势
- `similarity/stage_scores`：综合相似度和分层得分
- `fwd_returns`：后续5/10/20/60日收益

## 7. 接口调用

`POST /api/pattern/retrieval/query`，后续接 `POST /api/pattern/strategy/evaluate`

## 8. 页面状态

- **初始**：展示说明和输入框
- **请求中**：按钮禁用，日志面板显示请求进度
- **成功**：展示K线图和相似列表
- **无结果**：展示原因：样本不足/参考池无数据
- **错误**：页面和控制台同时输出错误

## 9. 验收标准

- [x] 必须显示相似股票代码和名称
- [x] K线不能为空，空数据必须报错
- [x] 相似片段可单独显隐
- [x] 前瞻走势可单独显隐
- [x] 点击按钮一定有页面日志
