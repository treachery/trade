# BE-000 项目模块骨架 — 实现文档

## 1. 模块划分

```
trade/
├── pattern_matching/     # 核心算法模块
│   ├── __init__.py       # 统一导出
│   ├── models.py         # 业务对象（BE-001）
│   ├── signals.py        # 信号侦测（BE-021）
│   ├── slicer.py         # K线切片+特征（BE-020/BE-030/BE-031）
│   ├── retrieval.py      # 相似检索（BE-032）
│   ├── evaluator.py      # 策略评估（BE-040）
│   ├── data_lake.py      # 批量加载（BE-012）
│   └── universe.py       # 股票池管理（BE-010）
├── simulation/            # Walk-Forward 模块
│   ├── __init__.py
│   ├── time_gate.py       # Point-in-Time 门控（BE-002）
│   ├── wf_engine.py       # WF 引擎（BE-051）
│   ├── rebalance.py       # 持仓再评估（BE-062）
│   └── reporter.py        # 绩效报告（BE-052）
├── governance/            # 治理模块
│   ├── __init__.py
│   ├── tracking.py        # 后验跟踪（BE-080）
│   ├── drift_monitor.py   # 漂移监控（BE-081）
│   └── lifecycle.py       # 策略生命周期（BE-082）
└── app.py                 # 仅保留路由/任务入口
```

## 2. 模块间依赖关系

```mermaid
flowchart TD
    app[app.py 路由层] --> universe[universe.py 股票池]
    app --> signals[signals.py 信号]
    app --> retrieval[retrieval.py 检索]
    app --> evaluator[evaluator.py 评估]

    retrieval --> slaves[slicer.py 切片+特征]
    retrieval --> data_lake[data_lake.py 批量加载]
    signals --> slaves
    data_lake --> backtest_data[backtest/data.py load_kline]

    universe --> backtest_scanner[backtest/scanner.py 成分股]

    app --> simulation[simulation/]
    app --> governance[governance/]
```

## 3. 时序逻辑

```mermaid
sequenceDiagram
    participant FE as 前端
    participant API as app.py
    participant Universe as universe.py
    participant Retrieval as retrieval.py
    participant Slicer as slicer.py
    participant DataLake as data_lake.py
    participant Backtest as backtest/data.py

    FE->>API: POST /api/pattern/retrieval/query
    API->>Retrieval: analyze_symbol(symbol, reference_symbols)
    Retrieval->>Backtest: load_kline(query_symbol)
    Retrieval->>DataLake: batch_load(reference_symbols)
    DataLake-->>Backtest: load_kline(each_symbol)
    Backtest-->>DataLake: DataFrame[]
    DataLake-->>Retrieval: {symbol: DataFrame}
    Retrieval->>Slicer: build_multi_stock_windows()
    Slicer-->>Retrieval: [{window}] + norm_stats
    Retrieval->>Slicer: extract_features(query)
    Slicer-->>Retrieval: query_features
    Retrieval-->>API: {query_kline, similar_windows, stages}
    API-->>FE: JSON response
```
