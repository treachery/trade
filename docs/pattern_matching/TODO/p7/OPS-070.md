# OPS-070 每日盘后任务编排

- **类型**：生产化
- **优先级**：P7
- **状态**：待办

---

## 1. 需求目标

每日15:30后自动运行数据更新、信号扫描、检索、评估和再平衡。

## 2. 需求范围

- 任务编排
- 异常捕获
- 结果落库/缓存
- 健康检查
- 前后端日志

## 3. 依赖关系

- `BE-062`

## 4. 示例图 / 流程图

```mermaid
flowchart TD
  schedule[15:30 Scheduler] --> data[数据更新]
  data --> signal[信号扫描]
  signal --> retrieval[相似检索]
  retrieval --> eval[策略评估]
  eval --> rebalance[持仓再评估]
  rebalance --> report[报告推送]
  report --> monitor[健康检查/日志]
```

## 7. 验收标准

- [ ] 任务失败不会静默
- [ ] 日志包含 query_id/耗时/结果数
- [ ] 健康接口可返回最近运行状态
