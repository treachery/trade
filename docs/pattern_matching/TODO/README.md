# 动态相似K线策略系统 TODO 拆解

来源：`docs/pattern_matching/quant_strategy_system_master_plan.md`。  
目标：把总体设计拆成可独立开发、可验收、可排序的前端/后端需求单，并标注依赖关系。

> 约定：
> - `BE-*`：后端/数据/算法/任务需求
> - `FE-*`：前端/交互/可视化需求
> - `QA-*`：验证/测试/质量需求
> - `OPS-*`：生产化/调度/监控需求
> - 依赖字段中的编号必须先完成，或至少提供 Mock/桩接口

---

## 0. 总体依赖顺序

```text
P0 基础规范与数据边界
  ↓
P1 数据湖 / 股票池 / Point-in-Time 数据访问
  ↓
P2 特征工厂 + 信号事件体系
  ↓
P3 K线片段库 + 逐级递进相似检索
  ↓
P4 相似样本策略评估 + 单标的研究页
  ↓
P5 Walk-Forward 样本外验证
  ↓
P6 组合再平衡 / 持仓再评估
  ↓
P7 盘后自动化 / 推送 / 监控
  ↓
P8 策略治理 / 后验跟踪 / 漂移降权
```

## P0：基础规范与数据边界

### BE-000 项目模块骨架整理

- **类型**：后端
- **目标**：按文档 9.3 建立清晰目录，避免把研究代码散落在 `app.py`。
- **范围**：
  - 新增 `pattern_matching/`：`features.py`、`signals.py`、`slicer.py`、`retrieval.py`、`evaluator.py`
  - 新增 `simulation/`：`time_gate.py`、`wf_engine.py`、`rebalance.py`、`reporter.py`
  - 新增 `governance/`：`tracking.py`、`drift_monitor.py`、`lifecycle.py`
  - `app.py` 只保留路由和任务入口
- **依赖**：无
- **验收标准**：
  - 所有模块可独立 import
  - 不破坏现有 `/`、`/trade-notify`、`/api/backtest`、`/api/optimize`

### BE-001 统一业务对象与数据模型定义

- **类型**：后端
- **目标**：先定义实体，后续接口和存储都围绕统一结构。
- **范围**：
  - 定义 `Security`、`DailyBar`、`DailyFeature`、`SignalEvent`、`PatternWindow`、`StrategyCandidate`、`StrategyEvaluation`
  - 定义 `UniverseConfig`、`WFSimulationRun`、`TradeDetail`、`PortfolioSnapshot`
  - 可先用 `dataclass` / `TypedDict`，后续再迁移 DB ORM
- **依赖**：BE-000
- **验收标准**：
  - 字段覆盖文档第 7 章核心表
  - 支持 JSON 序列化给前端展示

### BE-002 Point-in-Time 规则与泄漏检查规范

- **类型**：后端/质量
- **目标**：所有后续模块都必须知道“模拟日 D 能看到什么”。
- **范围**：
  - 实现 `time_gate.py` 的 `as_of_date` 数据过滤
  - 明确样本前瞻窗口闭合规则：`anchor_date + forward_days <= as_of_date`
  - 提供 `assert_no_future_data()` 自检函数
- **依赖**：BE-001
- **验收标准**：
  - 任意检索/评估结果可输出其 `as_of_date`
  - 使用未来 K 线时测试必须失败

### BE-003 A股交易约束规范实现

- **类型**：后端
- **目标**：为 Walk-Forward 和组合模拟提前实现交易可行性约束。
- **范围**：
  - T+1、整数手、涨跌停、停牌、佣金、印花税、滑点
  - 输出 `OrderResult(skipped, reason, cost, executed_qty)`
- **依赖**：BE-001、BE-002
- **验收标准**：
  - 可单测覆盖：涨停无法买入、跌停无法卖出、当日买入不可当日卖出、买入数量按 100 股取整

### QA-000 基础回归测试清单

- **类型**：质量
- **目标**：每次开发前后必须验证旧平台不坏。
- **范围**：
  - `/` 页面可打开
  - `/api/backtest` 默认参数返回 ok
  - `/trade-notify` 页面可打开
  - `/api/notify/symbols` 可返回宽基指数
- **依赖**：BE-000
- **验收标准**：
  - 提供脚本 `scripts/smoke_test.py` 或等价命令

## P1：数据湖、股票池与缓存

### BE-010 证券主数据与股票池管理

- **类型**：后端
- **目标**：区分“交易候选池”和“研究参考池”。
- **范围**：
  - 统一股票池配置：`trading_universe`、`reference_universe`
  - 复用 `backtest/scanner.py` 获取 A500 / SP500 / 港股大市值
  - 支持用户自定义代码列表
- **依赖**：BE-001
- **验收标准**：
  - API 可返回股票池 `{symbol, name, market, source}`
  - 交易池和参考池互不混淆

### BE-011 历史成分股快照存储

- **类型**：后端/数据
- **目标**：避免幸存者偏差。
- **范围**：
  - 存储 `index_membership` 快照：`index_code, symbol, effective_from, effective_to, weight, source`
  - 初期可用 JSON/CSV，后续 Parquet/DB
  - 提供按日期查询 `get_members(index_code, as_of_date)`
- **依赖**：BE-010、BE-002
- **验收标准**：
  - 查询历史日期时不默认使用当前成分股
  - 无历史数据时显式降级并给出警告

### BE-012 日K数据湖与增量更新

- **类型**：后端/数据
- **目标**：把 `load_kline()` 的缓存能力提升为可批量研究的数据湖。
- **范围**：
  - 标准化字段：`date/open/high/low/close/volume/amount/adjusted_*`
  - 按 `symbol/year` 分区存储 Parquet 或 CSV
  - 支持批量拉取与增量刷新
- **依赖**：BE-010
- **验收标准**：
  - 对 50 只股票可批量更新并缓存
  - 缓存命中时不联网

### FE-010 股票池配置组件

- **类型**：前端
- **目标**：让用户配置交易池/参考池。
- **范围**：
  - 支持选择 A500 / SP500 / 港股大市值 / 自定义代码
  - 清楚标注：交易池只用于买卖，参考池只用于学习
  - 展示股票数量和加载状态
- **依赖**：BE-010
- **验收标准**：
  - 用户能保存一组交易池和一组参考池配置

## P2：特征工厂与信号事件体系

### BE-020 日频特征工厂

- **类型**：后端/算法
- **目标**：将原始 OHLCV 转为统一特征快照。
- **范围**：
  - 收益：1d/5d/10d/20d/60d
  - 波动：20d/60d volatility、ATR14
  - 趋势：MA5/10/20/60/120、MA20/60斜率
  - 动量：RSI14、MACD、布林带
  - 量能：volume_zscore_20d、volume_ratio_5d
  - 环境：market_regime、volatility_regime、52w高低距离
- **依赖**：BE-012
- **验收标准**：
  - 任意 `symbol + date` 可查到完整特征快照
  - 前 N 日不足时返回 null，不得偷看未来

### BE-021 独立信号事件引擎

- **类型**：后端/算法
- **目标**：把现有扫描逻辑升级为可审计信号事件。
- **范围**：
  - 实现文档 5.1 的 10–20 个买入/卖出信号
  - 输出 `event_id, symbol, signal_date, signal_type, direction, strength, valid_next_trade_date, valid_until, decay_rate`
  - 支持信号生命周期与衰减
- **依赖**：BE-020、BE-002
- **验收标准**：
  - 每个信号说明触发条件和使用的特征快照
  - T 日收盘确认，T+1 才可执行

### BE-022 信号扫描 API

- **类型**：后端/API
- **目标**：给前端和自动任务提供统一信号入口。
- **接口建议**：
  - `POST /api/pattern/signals/scan`
  - 入参：`trading_universe_id | symbols, as_of_date, signal_types`
  - 出参：`signals[], feature_snapshot, diagnostics`
- **依赖**：BE-021、BE-010
- **验收标准**：
  - 支持批量扫描
  - 返回失败股票列表和失败原因

### FE-020 信号事件列表与详情

- **类型**：前端
- **目标**：展示哪些股票触发了什么信号，而不是只给结论。
- **范围**：
  - 信号表格：代码、名称、日期、方向、类型、强度、有效期、衰减状态
  - 信号详情：触发条件、特征快照、T+1执行提示
- **依赖**：BE-022
- **验收标准**：
  - 用户能从信号跳转到相似K线研究

## P3：K线片段库与递进相似检索

### BE-030 K线片段切片器

- **类型**：后端/算法
- **目标**：构建可跨股票检索的历史片段库。
- **范围**：
  - 对参考池全部股票按 `250日窗口 / 20日步长` 切片
  - 每段包含：`window_id, symbol, anchor_date, lookback_days, forward_days, feature_version, metadata`
  - 严格标注 `is_forward_window_complete`
- **依赖**：BE-012、BE-020、BE-002
- **验收标准**：
  - 片段必须带股票代码和名称
  - 对任意 `as_of_date` 能过滤掉未闭合前瞻窗口

### BE-031 片段标准化与特征向量构建

- **类型**：后端/算法
- **目标**：让不同价位、不同波动率的股票可比较。
- **范围**：
  - 对数收益率序列
  - z-score 标准化
  - t20/t40/t60/env 分层特征向量
  - 记录 `feature_version` 和权重配置
- **依赖**：BE-030
- **验收标准**：
  - 前端能看到每层使用的特征说明
  - 同一片段可复现相同向量

### BE-032 逐级递进相似检索引擎

- **类型**：后端/算法
- **目标**：按文档执行 `t-20 → t-40 → t-60 → env` 递进检索。
- **范围**：
  - t20：全片段库 → Top100
  - t40：Top100 → Top50
  - t60：Top50 → Top10
  - env：Top10 重排
  - 每个候选必须包含 `symbol, name, anchor_date, similarity, stage_scores, kline_before, kline_after`
- **依赖**：BE-031
- **验收标准**：
  - 结果必须列出“相似标的代码/名称 + 相似时间段”
  - 能追溯每一层输入/输出候选数和得分
  - 不允许只在同一标的自身历史里找相似片段，默认必须跨参考池

### BE-033 相似检索 API

- **类型**：后端/API
- **目标**：给研究页提供完整相似K线数据。
- **接口建议**：
  - `POST /api/pattern/retrieval/query`
  - 入参：`symbol, as_of_date, reference_universe_id, top_k, lookback_days, forward_days`
  - 出参：`query_kline, similar_windows[], stage_logs, diagnostics`
- **依赖**：BE-032、BE-010
- **验收标准**：
  - `similar_windows[]` 至少包含：`symbol, name, anchor_date, kline_before, kline_after, similarity`
  - K线图数据字段符合 ECharts candlestick 格式

### FE-030 相似K线研究页 MVP

- **类型**：前端
- **目标**：明确展示“当前K线”和“相似K线到底是谁”。
- **范围**：
  - 当前标的当前窗口 K 线图
  - 右侧相似片段列表：`股票代码/名称 + 相似时间段 + 相似度 + 后续收益`
  - 勾选/取消某个相似片段是否叠加到图上
  - 点击相似片段展示其后半段历史走势
- **依赖**：BE-033
- **验收标准**：
  - 页面必须显示相似标的代码和名称
  - 每个相似片段可单独显隐
  - 后半段走势与前半段用不同线型/颜色区分

### FE-031 递进检索链路可解释面板

- **类型**：前端
- **目标**：让用户知道 Top10 怎么来的。
- **范围**：
  - 展示 t20/t40/t60/env 每层输入输出数量
  - 每层候选 Top 列表与得分
  - 展示特征权重和距离指标
- **依赖**：BE-033
- **验收标准**：
  - 用户可展开某层查看候选池收缩过程

### QA-030 相似检索验收用例

- **类型**：质量
- **目标**：防止“只显示空K线/没有股票代码/只在自身历史找相似”的低质量实现。
- **范围**：
  - 测试返回至少 N 个不同 `symbol` 的相似片段
  - 测试每个片段有 `kline_before` 和 `kline_after`
  - 测试页面点击“开始分析”后必须出现日志或错误提示
- **依赖**：BE-033、FE-030
- **验收标准**：
  - 自动化脚本可验证 JSON 字段完整性
  - 前端 smoke test 可验证按钮点击触发网络请求

## P4：策略评估与单信号研究报告

### BE-040 候选策略库标准化

- **类型**：后端/算法
- **目标**：把策略定义从“随手写规则”变成可审计配置。
- **范围**：
  - 直接入场：DIR-5/10/20
  - 确认入场：CON-UP/CON-NB/CON-VOL
  - 回踩入场：PB-MA10/PB-BO/PB-COOL
  - 风控模板：SL、ATR、TIME、TRAIL
  - 每个策略包含 `entry_rule, exit_rule, risk_rule, parameter_set, version`
- **依赖**：BE-001
- **验收标准**：
  - 每个策略可序列化并显示给前端
  - 禁止无版本号策略参与评估

### BE-041 相似样本策略评估引擎

- **类型**：后端/算法
- **目标**：只在相似片段后半段上评估候选策略。
- **范围**：
  - 计算文档 6.2 指标：平均/中位数/胜率/盈亏比/最差20%/最大回撤/夏普/卡玛/Bootstrap
  - 支持样本数置信度：LOW/MED/HIGH
  - 防止少量样本过拟合
- **依赖**：BE-032、BE-040
- **验收标准**：
  - 样本数 <30 时必须标注“低置信度，仅供参考”
  - 不能只按平均收益排序

### BE-042 策略评估 API

- **类型**：后端/API
- **目标**：给前端展示策略排名和收益分布。
- **接口建议**：
  - `POST /api/pattern/strategy/evaluate`
  - 入参：`query_id | similar_windows, strategy_ids`
  - 出参：`strategy_evaluations[], best_strategy, distribution_data`
- **依赖**：BE-041
- **验收标准**：
  - 返回每个策略的指标和收益样本
  - 返回最佳策略不是硬编码

### FE-040 策略评估排名面板

- **类型**：前端
- **目标**：展示每种策略在相似情境下表现。
- **范围**：
  - 策略排名表：策略ID、名称、样本数、中位数收益、胜率、尾部风险、Bootstrap、稳健评分
  - 收益分布图：箱线图/小提琴图/直方图
  - 最佳策略卡片：推荐理由 + 置信度 + 风险提示
- **依赖**：BE-042
- **验收标准**：
  - 明确展示样本数和置信度
  - 风险指标不可隐藏

## P5：Walk-Forward 样本外验证

### BE-050 Walk-Forward 运行配置

- **类型**：后端
- **目标**：用户可配置样本外验证参数。
- **范围**：
  - 起止日期、交易池、参考池、仓位上限、初始资金、基准、成本参数、成交假设
  - 生成 `run_id` 和配置快照
- **依赖**：BE-010、BE-002、BE-003
- **验收标准**：
  - 每次运行配置可复现

### BE-051 时间门控逐日模拟引擎

- **类型**：后端/算法
- **目标**：逐日模拟真实决策，不用未来数据。
- **范围**：
  - 每个模拟日 D：更新数据 → 信号扫描 → 相似检索 → 策略评估 → 决策 → 次日执行
  - 所有样本必须通过前瞻窗口闭合校验
- **依赖**：BE-022、BE-033、BE-042、BE-050
- **验收标准**：
  - 任意一天的决策依据可重放
  - 空仓是合法动作

### BE-052 模拟交易与组合快照记录

- **类型**：后端
- **目标**：记录逐笔交易和每日组合状态。
- **范围**：
  - `simulation_trade_details`
  - `daily_portfolio_snapshots`
  - 交易成本、滑点、印花税
- **依赖**：BE-051、BE-003
- **验收标准**：
  - 每笔交易可追溯至信号事件、相似检索、策略版本

### FE-050 Walk-Forward 配置页

- **类型**：前端
- **目标**：用户可启动样本外验证。
- **范围**：
  - 配置区间、交易池/参考池、仓位、初始资金、成本
  - 运行状态和进度
- **依赖**：BE-050
- **验收标准**：
  - 提交配置后返回 `run_id`

### FE-051 Walk-Forward 报告页

- **类型**：前端
- **目标**：可视化样本外结果。
- **范围**：
  - 净值曲线 + 基准对比
  - 回撤曲线
  - 逐笔交易表
  - 每日决策日志
  - 指标卡：总收益、年化、最大回撤、胜率、换手率、夏普、卡玛
- **依赖**：BE-052
- **验收标准**：
  - 每笔交易能点开查看“为什么买/卖/换仓”

### QA-050 Walk-Forward 数据泄漏测试

- **类型**：质量
- **目标**：保证样本外验证可信。
- **范围**：
  - 测试 D 日无法访问 D+1 K线
  - 测试未闭合前瞻窗口不可参与策略评估
  - 测试信号确认日与执行日错开
- **依赖**：BE-051
- **验收标准**：
  - 泄漏测试失败时 CI/脚本必须报错

## P6：持仓再评估与组合再平衡

### BE-060 持仓性价比评分

- **类型**：后端/算法
- **目标**：判断当前持仓是否值得继续持有。
- **范围**：
  - 预期收益 × 置信度 × 信号持续性 × 流动性
  - 扣除下行风险、信号衰减、机会成本、换手成本
- **依赖**：BE-041、BE-052
- **验收标准**：
  - 输出 `hold_score, expected_return, expected_drawdown, risk_action`

### BE-061 形态漂移检测与重检索

- **类型**：后端/算法
- **目标**：入场后持续监控实际走势是否偏离参考片段。
- **范围**：
  - 实际走势 vs 参考片段后半段拟合度
  - 滚动相关性/残差/DTW 距离
  - 超阈值触发重检索和策略重评估
- **依赖**：BE-032、BE-060
- **验收标准**：
  - 每次漂移检测有日志和评分
  - 重检索结果保留版本

### BE-062 再平衡决策引擎

- **类型**：后端/算法
- **目标**：在持仓、新机会、现金之间做每日动作。
- **范围**：
  - hold / switch / clear / new_buy / cash
  - 换仓阈值：新机会分数 > 持仓分数 + 成本阈值
  - TopN 仓位管理
- **依赖**：BE-060、BE-061、BE-003
- **验收标准**：
  - 决策原因可解释
  - 现金为合法选择

### FE-060 持仓再评估页面

- **类型**：前端
- **目标**：展示每日持仓状态和动作建议。
- **范围**：
  - 持仓列表：持仓天数、入场信号、当前评分、漂移评分、风险动作
  - 对比新机会列表
  - 决策原因时间线
- **依赖**：BE-062
- **验收标准**：
  - 用户能看懂为什么继续持有/换仓/清仓

## MVP 必须满足的底线

1. 相似结果必须包含不同股票的 `symbol/name/anchor_date`，不能只显示“相似1/相似2”。
2. 当前标的 K 线和相似片段 K 线必须都是真实 OHLC，不允许空 K 线。
3. 每个相似片段必须能单独勾选显隐，并可展示后半段走势。
4. 页面点击按钮必须有前端日志；后端必须有请求日志。
5. 检索必须遵守 Point-in-Time：当前日期之后的数据不可见，未闭合前瞻窗口不可用。
6. 策略评估必须展示样本数、置信度、尾部风险，不能只给“推荐买入”。
