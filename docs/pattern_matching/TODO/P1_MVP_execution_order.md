# P1 MVP 执行顺序清单

> 这是第一阶段实际开发顺序，目标是先把“跨股票相似K线 + 可视化 + 策略评估”做成可用闭环。

## 第 1 批：后端基础

1. `BE-000` 模块骨架整理
2. `BE-001` 业务对象与数据模型
3. `BE-002` Point-in-Time 规则
4. `BE-010` 股票池管理
5. `BE-012` 日K数据湖与增量缓存

## 第 2 批：算法基础

6. `BE-020` 日频特征工厂
7. `BE-021` 信号事件引擎
8. `BE-022` 信号扫描 API

## 第 3 批：相似K线核心

9. `BE-030` K线片段切片器
10. `BE-031` 片段标准化与特征向量
11. `BE-032` 跨股票逐级递进检索引擎
12. `BE-033` 相似检索 API

## 第 4 批：前端可视化

13. `FE-030` 相似K线研究页 MVP
14. `FE-031` 递进检索链路可解释面板
15. `QA-030` 相似检索验收用例

## 第 5 批：策略评估

16. `BE-040` 候选策略库标准化
17. `BE-041` 相似样本策略评估引擎
18. `BE-042` 策略评估 API
19. `FE-040` 策略评估排名面板

## 每批完成后的验收动作

- 每批必须跑 `QA-000` 基础回归
- 第 3 批后必须能返回至少 5 个不同 `symbol` 的相似片段
- 第 4 批后页面必须展示：当前K线、相似股票代码/名称、相似时间段、叠加开关、后半段走势
- 第 5 批后必须展示：策略排名、样本数、置信度、尾部风险、Bootstrap CI
