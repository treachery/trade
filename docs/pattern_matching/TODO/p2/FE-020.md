# FE-020 信号事件列表与详情

- **类型**：前端
- **优先级**：P2
- **状态**：待办

---

## 1. 需求目标

展示扫描到的信号事件，并允许跳转到相似K线研究。

## 2. 页面 / 组件范围

- 信号列表页
- 信号详情抽屉
- 跳转相似研究按钮

## 3. 按钮与交互

- **扫描信号**：调用批量扫描API
- **筛选**：按方向/强度/日期/市场筛选
- **查看详情**：打开信号触发条件和特征快照
- **研究相似K线**：跳转 FE-030 并带上 symbol/as_of_date/event_id

## 4. 展示数据

- `symbol/name`：标的代码和名称
- `signal_type`：信号类型
- `strength`：信号强度
- `valid_next_trade_date`：可执行日
- `feature_snapshot`：触发时特征

## 5. 依赖关系

- `BE-022`

## 6. 示例图 / 页面草图

```text
+----------------------------------------------------+
| 信号事件                                           |
| [扫描信号] [方向:全部] [强度:强以上] [日期]          |
|----------------------------------------------------|
| 代码  名称  日期  方向  信号        强度  操作       |
|600519 茅台 06-29 买入 突破20日新高 强   [详情][研究]|
+----------------------------------------------------+
```

## 7. 接口调用

`POST /api/pattern/signals/scan`

## 8. 页面状态

- **扫描中**：显示进度和当前扫描股票
- **无信号**：显示空状态而不是空白
- **失败**：展示 failed 列表

## 9. 验收标准

- [ ] 列表必须显示股票代码和名称
- [ ] 点击研究能带参进入相似K线页
